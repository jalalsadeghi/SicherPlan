# AI-65 — Dynamic Lookup and Tenant Catalog Explanation Tools

## Purpose

Static field definitions are not enough. SicherPlan has many lookup/catalog/status values that can be tenant-specific, API-driven, or seeded by configuration. The Assistant must answer questions about lookup fields and options without leaking unauthorized tenant data.

Examples:

```text
was bedeutet Klassifizierung
was ist Ranking
was bedeutet Freigabestatus
was bedeutet release_ready
what does workforce scope mixed mean
what does customer visible mean
```

This task adds runtime lookup/catalog explanation tools and integrates them with RAG.

---

## Non-Negotiable Rules

1. Do not index private operational records as global knowledge.
2. Tenant-specific lookup labels may be read only through authorized tools.
3. Do not leak values the user cannot access.
4. Final answer must be OpenAI-generated from retrieved/tool facts.

### Step 3 — Static + Dynamic Strategy

Use both:

1. Static dictionary from AI-63 for common enum/status values.
2. Dynamic authorized API/tool lookups for tenant-specific catalogs.

If dynamic values are unavailable:
- return static explanation if available;
- say that tenant-specific labels may differ.

### Step 4 — Normalize Lookup Queries

Support:
- value code: `release_ready`;
- localized label: `Freigabebereit`;
- field label: `Freigabestatus`;
- option label: `Mixed`;
- i18n key: `customerAdmin.status.active`.

### Step 5 — Integrate With Retrieval Planner

If classifier detects lookup/status question:
- call lookup dictionary/search tool;
- add result to grounding context;
- link related page if known;
- if ambiguous, pass candidates to OpenAI for clarification.
