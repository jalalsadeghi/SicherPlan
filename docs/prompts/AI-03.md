## Goal

Add the initial IAM permission baseline for the SicherPlan AI Assistant. The assistant must be available only to users with explicit assistant permissions. This task prepares permission keys and role mappings according to the existing SicherPlan IAM seed/convention.

This task must not implement the assistant API, tool registry, database assistant tables, frontend widget, OpenAI wrapper, or diagnostics. It only adds/aligns permission definitions and, where appropriate, small policy constants/helpers needed for future tasks.

## Hard Rules

1. Follow the existing IAM permission seed and role-mapping style.
2. Do not create a parallel permission system.
3. Do not grant broad access blindly.
4. Do not change unrelated permissions.
5. Do not implement assistant endpoints.
6. Do not implement assistant tools.
7. Do not add assistant database conversation/messaging tables.
8. Do not add frontend code.
9. Do not create write permissions for assistant business actions.
10. Keep the assistant baseline read-only.
11. If role keys differ from sprint names, map carefully and report exact mappings.
12. If uncertain about a role mapping, choose the safer narrower mapping and report it.

## Permission Meaning

### `assistant.chat.access`

Allows the user to see and use the assistant chat widget if `SP_AI_ENABLED=true`.

This permission does **not** allow diagnostics by itself.

### `assistant.diagnostics.read`

Allows the assistant service to run read-only operational diagnostic tools when the user also has the underlying domain permissions.

This permission does **not** bypass domain permissions. Example:

```text
assistant.diagnostics.read + planning assignment read permission
```

must both be required for future assignment diagnostics.

### `assistant.feedback.write`

Allows the user to submit helpful/not-helpful feedback for accessible assistant messages.

### `assistant.knowledge.read`

Allows the assistant to use indexed platform documentation for how-to answers. This permission does not expose operational data.

### `assistant.knowledge.reindex`

Allows a trusted admin/internal user to trigger knowledge reindexing. This must be restricted.

### `assistant.admin`

Administrative assistant permission for future support/admin capabilities. This must be restricted to platform admin or equivalent.
