# AI-63 — Platform Field & Lookup Knowledge Corpus for SicherPlan Assistant

## Purpose

The Assistant currently treats short field-label questions like:

```text
was bedeutet Rechtlicher Name
```

as out-of-scope unless the user explicitly adds `im SicherPlan`.

This is wrong for an in-app assistant. If an authenticated user asks about a visible field, form label, lookup, status, catalog option, tab, workspace, action label, or column label, the Assistant must assume the question is about SicherPlan unless there is strong evidence otherwise.

This task builds a comprehensive **Field & Lookup Knowledge Corpus** for the AI Assistant. It must extract, normalize, index, and expose all known platform fields and lookup/catalog terms so RAG can answer field meaning questions accurately.

---

## Non-Negotiable Rule

Do **not** create canned final answers.

Create retrievable field/lookup facts with source basis. OpenAI still generates the final answer from those facts.

---

## Problem Example

Current behavior:

```text
User: was bedeutet Rechtlicher Name
Assistant: Ich bin nur dafür vorgesehen...
```

Expected behavior:

```text
Assistant understands "Rechtlicher Name" as a SicherPlan field label, likely customer legal name, especially when the current route is Customers or the label is unique in the field dictionary.
```

The answer may differ depending on context:

- On Customers page: legal name of the customer/legal entity used for official documents, contracts, invoices, and billing identity.
- If ambiguous: explain likely meaning and mention possible related places, then ask for page/context if needed.

### Step 5 — Source Basis and Confidence

Each extracted field must have a confidence level:

```text
high: label + form binding + API/schema field found
medium: label + likely field key found
low: label only or schema only
```

The Assistant can answer high/medium confidently. Low confidence should include uncertainty.

### Step 6 — Index as Knowledge Chunks

Index field and lookup definitions into the Assistant knowledge repository.

Source types:

```text

Related fields: Customer number, Name, External reference...
```
