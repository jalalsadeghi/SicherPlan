# AI-70 — Fix CI Field/Lookup Corpus Verification Dependency Bootstrap

## Purpose

GitHub Actions fails in the step:

```text
Verify committed field/lookup corpus artifact is current
```

with:

```text
ModuleNotFoundError: No module named 'sqlalchemy'
```

The failing call is:

```bash
python -m app.modules.assistant.field_dictionary_export ...
```

The import chain is:

```text
field_dictionary_export
  -> app.modules.assistant.field_dictionary
  -> app.modules.assistant.page_help_seed
  -> from sqlalchemy import select
```

The repository already declares `sqlalchemy>=2.0` in `backend/pyproject.toml`, so the root cause is not a missing project dependency declaration. The root cause is that the GitHub Actions verification step runs the backend module before installing the backend package/dependencies in that job.

This task must fix the CI bootstrap and also make the export tool less fragile where reasonable.

---

## Non-Negotiable Requirements

1. Do not remove SQLAlchemy from normal backend dependencies.
2. Do not remove the field/lookup corpus verification.
3. Do not skip the stale-artifact check.
4. Do not make CI pass by ignoring the export failure.
5. The verification step must run in an environment where backend dependencies are installed.
6. The command must remain deterministic and suitable for CI.
