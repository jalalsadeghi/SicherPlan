# Codex Prompt — US-15-T1

## Task title

**Build subcontractor portal authentication, role scope, and navigation**

## Objective

Create the subcontractor self-service entry point so approved partner users can log in, land in a portal-safe navigation shell, and operate strictly within their subcontractor scope.

## Source context

- Updated proposal: section 4 role model includes subcontractor users; section 5.4 subcontractor collaboration portal; section 6.3 step 1 through step 4 of the subcontractor collaboration workflow.
- Implementation specification: `iam.user_account`, `iam.user_role_scope`, subcontractor-contact portal linkage, and the portal visibility chain in section 5.
- Uploaded Dienstplanmacher subcontractor source material: the own-login subcontractor flow shows that partner users prepare staff and confirm work through a dedicated login before or alongside operational execution.
- Sprint reference: `docs/sprint/sprint-05-subcontractor-management.md`.
- Cross-cutting rules: `AGENTS.md`.

## Dependencies

- `US-5-T1`, `US-5-T2`, and `US-5-T3` must already provide the core login/session/scope infrastructure.
- `US-13-T1` and `US-13-T4` should already provide portal-linked subcontractor contacts and partner-specific authorization rules.
- `US-3-T1`, `US-3-T3`, and `US-3-T4` should already provide the shared Vben shell patterns, theme tokens, and DE/EN localization baseline.

## Scope of work

- Build the subcontractor-portal login entry and authenticated shell by reusing the existing IAM/session foundation rather than creating a second auth mechanism.
- Implement route guards and page bootstrap logic that derive the effective subcontractor scope from the logged-in user and linked contact/role-scope data.
- Create the first portal navigation structure for company profile, workers, documents, released work, confirmations, and control/report sections, with feature-aware hiding where downstream data or pages do not exist yet.
- Ensure the portal layout clearly separates external partner navigation from internal admin navigation even if both are implemented inside the same web codebase.
- Localize all new portal labels, empty states, and auth-related texts in German and English, and reuse the shared theme tokens and Vben-compatible layout primitives.
- Add tests for login success/failure, route guarding, subcontractor-scope bootstrapping, and navigation visibility behavior.

## Preferred file targets

- Portal routes, layout shell, auth guards, menu generation, API clients, and locale resources under the web application path.
- Minimal backend auth/scope endpoints or DTO adjustments only where necessary to support the portal shell.
- Frontend tests around portal auth and guarded navigation.

## Hard constraints

- Reuse the approved IAM/session foundation; do not create a custom subcontractor-only login stack or duplicate token model.
- External users are tenant-scoped actors, not separate tenants. Every portal route and query must carry both tenant and subcontractor scope.
- Do not expose internal admin menus, broad tenant search, or unrelated customer/employee areas in the subcontractor portal shell.
- Navigation items for future released-work or finance views may appear only when backed by real routes or explicit feature placeholders; do not fake data-rich screens before their contracts exist.
- All new portal UI text must ship in German and English and follow the shared light/dark theme tokens.

## Expected implementation outputs

- A subcontractor portal shell with login, route guards, and scope-safe navigation.
- Portal bootstrap logic that resolves the effective subcontractor from the logged-in account and contact linkage.
- Frontend/backend test coverage for auth, scope, and navigation behavior.

## Non-goals

- Do not implement the deeper released-work or invoice-status views here; that is `US-15-T2`.
- Do not implement allocation workflows here; that is `US-15-T3`.
- Do not implement staff/data self-service screens here beyond what is needed for navigation and auth; that is `US-15-T4`.

## Verification checklist

- [ ] Approved subcontractor users can authenticate through the existing IAM/session stack and land in the portal shell.
- [ ] Portal navigation is restricted to the user's linked subcontractor scope.
- [ ] Internal/admin-only menus and broad tenant-search capabilities are not exposed in the partner portal.
- [ ] German is the default locale and English is available for all new portal auth/navigation text.
- [ ] Tests cover guarded routes and invalid-scope behavior.

- A concise list of files created or modified.
- A summary of key design decisions and assumptions.
- Verification evidence: build/lint/test commands and notable outputs.
- Open follow-ups or blockers that should be carried into the next task.
