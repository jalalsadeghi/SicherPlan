# AI-68 — CI, Docker, and Stage Verification for Assistant Field/Lookup Corpus

## Purpose

After AI-67 creates a portable generated field/lookup corpus artifact, this task ensures it is packaged, validated, and deployed correctly through Docker/GitHub Actions/Stage.

The Stage issue occurred because `backend/Dockerfile.stage` copies only `backend` into `/app/backend`, so runtime frontend source extraction is unavailable. With a generated backend artifact, Stage should work without copying the whole frontend source.

### Step 4 — Docker Image Smoke Test

Add a CI smoke test after backend image build:

```bash
docker run --rm <backend-image> python - <<'PY'
from app.modules.assistant.field_dictionary import (
    detect_field_or_lookup_signal,
    field_definition_counts_by_module,
)

print(field_definition_counts_by_module())
assert detect_field_or_lookup_signal("was bedeutet Vertragsreferenz") is not None
assert detect_field_or_lookup_signal("was bedeutet Rechtlicher Name") is not None
assert detect_field_or_lookup_signal("was bedeutet Apfelkuchen") is None
PY
```

Use the correct CI image tag.
