# AI-44 — German/English Hybrid Retrieval, Reranking, and Grounding Package Builder

## Purpose

Improve retrieval quality so German and English questions retrieve the correct SicherPlan sources and pass enough grounded context into the OpenAI prompt.

This task focuses on retrieval and grounding packaging, not on final-answer templates.

---

## Product Requirement

For every in-scope German or English question, the assistant must:

1. rewrite/expand the query with SicherPlan domain terms;
2. retrieve relevant chunks/facts/tools;
3. rerank sources based on intent, page, module, language, source type, and content quality;
4. reject or demote irrelevant shallow sources;
5. build a compact grounding package;
6. pass that package to OpenAI.

---

### Step 3 — Query Expansion

Add a query expansion function:

```python
expand_query_for_retrieval(query: str, language_code: str | None) -> ExpandedQuery
```

Output should include:

```json
{
  "original_query": "...",
  "detected_terms": ["contract", "document"],
  "expanded_terms_en": ["contract", "agreement", "document", "attachment"],
  "expanded_terms_de": ["Vertrag", "Dokument", "Anhang"],
  "likely_page_ids": ["PS-01", "C-01", "P-02"],
  "likely_module_keys": ["platform_services", "customers", "planning"]

```

### Step 4 — Retrieval Planner Improvements

For German/English questions, map intents:

```text
customer_create -> C-01
order_create -> P-02, C-01
customer_plan_create -> C-01, P-02, P-03
contract_or_document_register -> PS-01, C-01, P-02, S-01 when partner context
employee_assign_to_shift -> E-01, P-03, P-04, P-05
shift_release_to_app -> P-03, P-04, P-05, ES-01
```

Dashboard `F-02` should only be selected for dashboard questions.

Employee `E-01` should only be selected for employee/personnel/staff questions.

### Step 5 — Source Reranking

Add reranking rules:

Boost:
- content-bearing sources;
- exact workflow intent;
- exact page ID;
- exact module key;
- page_help_manifest for UI-specific questions;
- workflow_help for process questions;
- operational_handbook/user_manual for how-to questions;
- API export for endpoint/API questions.

Demote:
- dashboard unless dashboard is mentioned;
- employee sources unless employee/personnel is mentioned;
- shallow page route sources when richer chunks exist;
- sources with empty or tiny content;
- sources whose page/module conflicts with detected intent.

### Step 6 — Grounding Package Limits

Build a compact package:

```text
max_sources: configurable, default 10
max_chars_per_source: configurable, default 1200
max_total_grounding_chars: configurable, default 9000
```

Every source included in the prompt must have:
- `source_type`
- `source_name`
- `page_id`
- `module_key`
- `title`
- `content` or `facts`
- `score`
- `why_selected`
