# AI-72 — Platform UI Term, Concept, and Glossary Corpus for Assistant

## Purpose

The Assistant now handles many form fields, but it still rejects platform terms that are not modeled as form fields or lookups.

Failing example:

```text
User: was bedeutet Demand groups?
Assistant: Ich bin nur dafür vorgesehen, Fragen und Probleme zur SicherPlan-Plattform zu beantworten...
```

This term exists in the platform. In Planning Staffing / Shift Coverage, the i18n/messages include terms such as:

```text
demandGroupsTitle: "Demand Groups" / "Demand groups"
demandGroupsEmpty
demandGroupsSetupRequired
demandGroupEditorTitle
demandGroupCreateTitle
demandGroupEditTitle
demandGroupFunctionType
demandGroupQualificationType
demandGroupMinQty
demandGroupTargetQty
demandGroupMandatoryFlag
fieldsDemandGroup
```

The existing field/lookup corpus is too narrow. It primarily extracts form fields/lookups and misses many visible UI terms, section titles, tab labels, action labels, validation labels, empty-state messages, hints, and domain concepts.

This task creates a broader **Platform UI Term / Concept / Glossary Corpus** so the Assistant can answer questions about any visible platform term, not only fields.

---

## Non-Negotiable Requirements

1. Do not create canned final answers.
2. Do not add only one hardcoded keyword for `Demand groups`.
3. Generalize to all visible platform terms, labels, section titles, tabs, actions, hints, empty states, validation terms, and domain concepts.
4. Final answers must still be generated through the Assistant RAG/OpenAI path.
5. Unknown unrelated terms should remain out-of-scope.
6. Stage must work without frontend source files at runtime; generated artifact must be portable.
7. Terms must include source basis.

### Step 5 — Domain Meaning Extraction

For each UI term/concept, infer meaning from:

1. i18n hint/lead/setup texts;
2. page-help manifests;
3. workflow help;
4. backend schema field names;
5. domain service code names;
6. tests and smoke tests;
7. related labels.

For Demand Groups, definition must be based on existing source facts:
- a shift cannot be staffed until at least one demand group is created;
- demand groups contain function type, optional qualification, min quantity, target quantity, sort order, mandatory flag, and remark;
- staffing actions are blocked until a shift has at least one demand group.
