# AI-74 — Golden QA for Platform UI Terms and Domain Concepts

## Purpose

Prevent regressions for platform terms that are not simple form fields.

The Assistant must answer questions about visible UI terms and domain concepts such as:

```text
Demand groups
Staffing actions
Release-Gates
Override-Nachweise
Partnerfreigaben
Dispatch-Nachrichten
Coverage
Minimum staffing
Mandatory proofs
```

without requiring the user to explicitly add `im SicherPlan`, `im Shift coverage`, or the exact page name.

---

## Prompt for Codex

You are working on the SicherPlan repository.

Implement **AI-74 — Golden QA for Platform UI Terms and Domain Concepts**.

### Step 3 — Expected Criteria

For each known platform term:

1. Classified in-scope.
2. Correct intent:
   - `platform_term_meaning_question`
   - `ui_label_meaning_question`
   - `domain_concept_question`
3. Retrieves a platform term/concept source.
4. Retrieves related page/workflow facts where available.
5. Provider is called.
6. Answer language follows the user’s language.
7. Answer explains:
   - meaning;
   - where it appears;
   - why it matters;
   - what user can do next.
8. Source basis exists.
9. No out-of-scope refusal.
10. No fake routes or placeholder links.

### Step 4 — Demand Groups Regression

Add explicit regression:

```text
was bedeutet Demand groups
```

Expected:

```text
- in-scope;
- term key: planning.staffing.demand_groups;
- module: planning;
- related page: Staffing Coverage / /admin/planning-staffing;
- answer mentions that demand groups define staffing requirements/slots for a shift;
- answer mentions function type, optional qualification, minimum/target quantity, mandatory flag when available from sources;
- answer says staffing actions may require demand-group setup.
```
