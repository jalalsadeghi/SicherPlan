# AI-42 — German/English RAG Knowledge Corpus Ingestion and Chunking

## Purpose

Build a real, content-bearing RAG knowledge corpus for the SicherPlan AI Assistant, focused for now on **German and English** user questions.

The assistant must stop relying on shallow page hints or generic model knowledge. It must retrieve actual SicherPlan-specific documentation, page/workspace scope, workflow facts, API/page mappings, and UI/action facts before asking OpenAI to generate an answer.

This task focuses only on the **knowledge corpus and chunking layer**. It must not introduce canned final answers.

---

## Product Requirement

For German and English questions, the assistant must be able to retrieve content-bearing sources from the SicherPlan knowledge base.

Examples:

```text
Wie registriere ich einen neuen Vertrag?
How do I create a new customer?
How do I create a new order for a customer?
Wie weise ich einen Mitarbeiter einer Schicht zu?
How do I release a shift to the employee app?
```

The retrieved context must contain real details such as:

- relevant page/workspace;
- module ownership;
- workflow chain;
- verified UI action facts, if available;
- role/permission constraints;
- API family / endpoint family;
- source title and source type;
- concise content excerpt;
- source-basis metadata.

---

## Non-Negotiable Rule

Do **not** create prewritten final answers.

This task creates retrievable knowledge chunks and metadata. OpenAI must generate the final user-facing answer later from these chunks.

### Step 3 — Content-Bearing Chunk Contract

Each retrievable chunk must be content-bearing.

Minimum fields:

```json
{
  "source_type": "user_manual",
  "source_name": "SicherPlan Platform User Manual",
  "source_path": "docs/...",
  "source_hash": "...",
  "source_language": "en|de|mixed|unknown",
  "chunk_index": 1,
  "title": "C-01 — Customers Workspace",
  "content": "real text content, not only a page ID",
  "content_preview": "short safe preview",
  "page_id": "C-01",
  "module_key": "customers",
  "workflow_keys": ["customer_create", "customer_order_create"],
  "role_keys": [],
  "permission_keys": [],
  "api_families": ["customers"],
  "domain_terms": ["customer", "kunde", "customer master", "billing profile"],
  "language_aliases": ["customer", "Kunde", "Kunden", "customers"]
}
```

Do not index operational records such as real customers, employees, shifts, invoices, or private HR/payroll data into the documentation knowledge base.

### Step 4 — Chunking Strategy

Implement or improve chunking:

- target chunk length: 500–900 words or equivalent token-aware size;
- overlap: 80–150 words where useful;
- keep headings and page/workspace IDs attached to each chunk;
- split long docs by section/page/workflow, not arbitrary line count only;
- preserve page IDs such as `C-01`, `P-02`, `P-03`, `P-04`, `PS-01`;
- preserve API family names such as `customers`, `planning`, `platform-docs`;
- derive module key from page ID/API family when possible.

### Step 5 — Seed Page/Workflow Sources as Knowledge

Convert existing page route catalog, page help manifest, workflow help, and UI action catalog into retrievable knowledge chunks.

Example chunk:

```json
{
  "source_type": "page_help_manifest",
  "source_name": "Page Help Manifest",
  "title": "C-01 — Customers Workspace",
  "page_id": "C-01",
  "module_key": "customers",
  "content": "Customers Workspace is the tenant-scoped customer master-data and commercial control center. It covers customer registry/detail, contacts, billing profile, invoice parties, rate cards, surcharge rules, addresses, history, portal privacy, employee blocks, imports/exports, and vCard export."
}
```

Example workflow chunk:

```json
{
  "source_type": "workflow_help",
  "source_name": "Workflow Help",
  "title": "Customer order to report and invoice",
  "workflow_keys": ["customer_order_create", "customer_plan_create"],
  "page_id": "P-02",
  "module_key": "planning",
  "content": "Customer order/project starts in Orders & Planning Records, then creates planning records, shifts, assignments, dispatch outputs, actuals, timesheets, and invoices."
}
```
