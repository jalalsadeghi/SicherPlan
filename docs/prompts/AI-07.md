This task is **AI-07 — Structured Response Contract**.

## Goal

Define and implement the canonical backend response contract for all AI Assistant answers.

The goal is to make assistant responses predictable for frontend rendering and safe for future diagnostic, knowledge, navigation, and provider tasks. Every assistant answer must return a structured object containing the natural-language answer, language metadata, confidence, diagnosis/evidence, safe links, missing permissions, next steps, and out-of-scope status.

This task should not implement OpenAI, knowledge retrieval, tool registry, diagnostic tools, or frontend widget rendering yet. It only creates the shared response schema and integrates it into the existing assistant message endpoint in a minimal safe way.

## Hard Rules

1. Follow the existing backend schema, router, service, repository, and test conventions.
2. Do not call OpenAI in this task.
3. Do not add provider logic or mock provider orchestration in this task, except for a simple deterministic placeholder response if the current endpoint needs one.
4. Do not implement knowledge ingestion or retrieval.
5. Do not implement tool registry or diagnostic tools.
6. Do not implement page route catalog or navigation link generation yet.
7. Do not implement frontend components in this task.
8. Do not add business write actions.
9. Do not generate or execute SQL from user messages.
10. Do not expose secrets, raw auth context, JWTs, reset tokens, credentials, or stack traces in responses.
11. Do not expose HR-private or payroll-sensitive fields.
12. Preserve the conversation/message persistence behavior from AI-06.
13. Keep route context as non-authoritative metadata only.
14. If existing file names differ from this prompt, adapt to repository conventions and report the difference.

### Field Rules

#### `conversation_id`

- Required.
- Must match the conversation receiving the message.

#### `message_id`

- Required.
- Must refer to the stored assistant message.

#### `detected_language`

- Required but may be `unknown` until AI-08 implements language detection.
- In this task, default can be based on existing request locale or `unknown`.

#### `response_language`

- Required but may be `unknown` until AI-08.
- Must never be missing from the response object.

#### `answer`

- Required non-empty string.
- Must be safe for frontend rendering as plain text or markdown according to current UI conventions.
- Must not contain raw JSON dumps, stack traces, secrets, or tool payloads.

#### `scope`

Allowed values:

```text
platform
tenant
branch
mandate
customer
subcontractor
employee_self_service
unknown
```

Use `unknown` if scope cannot yet be determined safely.

#### `confidence`

Allowed values:

```text
low
medium
high
```

For placeholder responses before provider/tools exist, use `low`.

#### `out_of_scope`

- Required boolean.
- Must default to `false` in this task.
- AI-09 will implement real out-of-scope behavior.

#### `diagnosis`

- Required list.
- May be empty.
- Items must contain `finding`, `severity`, and `evidence`.
- Do not include raw sensitive records.

Severity values:

```text
info
warning
blocking
```

#### `links`

- Required list.
- May be empty.
- AI-17 will generate real safe links.
- In this task, do not invent links.

#### `missing_permissions`

- Required list.
- May be empty.
- Must not expose inaccessible record existence.

#### `next_steps`

- Required list.
- May be empty.

#### `tool_trace_id`

- Optional UUID/null.
- Must be null in this task unless existing tool-call audit exists.

Rules:

- Store only the structured assistant response.
- Do not store raw request auth context.
- Do not store secrets or full internal exception details.
- Keep user message content storage unchanged from AI-06.
