# AI-41 — RAG Corpus Coverage and Domain Retrieval for Contract, Customer, Order, and Planning Questions

## Purpose

Fix the specific RAG quality gap where questions about contracts, customers, orders, and customer planning receive generic answers or incorrect shallow sources.

The assistant must retrieve the correct SicherPlan knowledge for domain questions such as:

```text
Wie registriere ich einen neuen Vertrag?
چطور باید مشتری ثبت کنم؟
چطور میتونم یک پلن جدید برای مشتری ثبت کنم؟
I want to create a new order. How can I do it?
```

The final answer must still be generated by OpenAI, but the retrieved context must contain the correct SicherPlan-specific facts.

---

## Product Expectation

For the German question:

```text
Wie registriere ich einen neuen Vertrag?
```

The assistant should not pretend that there is a standalone "Contract" page unless the repository/docs verify it.

A good RAG-grounded answer should explain something like:

- In the current SicherPlan scope, contracts are handled as documents/attachments and linked to business entities rather than through a standalone "Contract" module, if that is what the docs confirm.
- Use `PS-01 Platform Services Workspace` for document create/version/link workflows, if verified.
- Use `C-01 Customers Workspace` when the contract belongs to a customer relationship, customer history, billing/commercial profile, or customer document.
- Use `P-02 Orders & Planning Records` when the contract/order document belongs to an order/project/planning record.
- Ask a clarifying question if the user means a customer contract, an order attachment, a billing agreement, or a subcontractor agreement.

Codex must verify these facts against the actual docs and code before implementing.

### Step 4 — Add Intent Detection for Contract/Document Registration

Add a retrieval intent:

```text
contract_or_document_register
```

Trigger examples:

```text
Wie registriere ich einen neuen Vertrag?
Wie lege ich einen Vertrag an?
چطوری یک قرارداد جدید ثبت کنم؟
How do I upload/register a contract?
How do I add a contract document?
```

Expected likely pages:

```text
PS-01
C-01
P-02
S-01 if subcontractor contract is implied
```

Expected behavior:

- If the question is ambiguous, retrieve context for all plausible areas and let OpenAI ask a targeted clarifying question.
- Do not choose dashboard `F-02`.
- Do not choose employees `E-01` unless employee contract/personnel file is explicitly mentioned.

### Step 5 — Improve Retrieval Ranking

For contract/customer/order/planning questions, retrieval must prefer:

1. Exact domain intent page IDs.
2. Matching module keys.
3. Chunks whose title/content includes translated synonyms.
4. Page-help/workflow facts over dashboard summaries.
5. Handbook/user manual chunks over generic page titles.
6. Current route context only if relevant.

Add negative scoring or demotion:

```text
F-02 dashboard should be demoted unless query mentions dashboard/KPI/landing page.
E-01 employees should be demoted unless query mentions employee/personnel/guard/HR.
```
