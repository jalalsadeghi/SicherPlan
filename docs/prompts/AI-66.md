### Step 3 — Expected Criteria

For each field/lookup question:

1. Classified in-scope.
2. Correct intent:
   - `field_meaning_question`
   - `lookup_meaning_question`
   - `status_meaning_question`
3. Retrieves field/lookup dictionary source.
4. Provider is called.
5. Answer language matches user language.
6. Explains:
   - what the field/lookup means;
   - where it appears;
   - how it is used;
   - whether exact tenant values may vary.
7. Source basis exists.
8. Does not return out-of-scope refusal.
9. Does not invent unsupported details.

### Step 4 — Special Regression

The following must never refuse:

```text
was bedeutet Rechtlicher Name
```

Expected field:
```text
customer.legal_name
```

Expected page/module:
```text
Customers Workspace / customers
```

Expected explanation concept:
```text
official/legal customer name used for formal documents, contracts, invoices, or legal identity
```

### Step 5 — Ambiguity Golden Tests

Add ambiguous examples:

```text

```

Expected:
- in-scope;
- answer should mention likely candidates or ask clarification;
- should prefer current route context if provided.
