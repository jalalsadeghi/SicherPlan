# AI-60 — Add Customer-Scoped Orders / Order Workspace to Assistant RAG Knowledge

## Purpose

The AI Assistant currently understands the older/canonical Operations & Planning route for orders, but it does not reliably understand the new customer-scoped order flow:

```text
Customers → selected customer → Orders tab → New order/Edit → Order workspace
```

This task adds the latest Customer Orders tab and Order Workspace flow to the Assistant’s RAG knowledge so users can ask questions such as:

```text
How can I create an order from the customer page?
Wie erstelle ich einen Auftrag direkt beim Kunden?
Wie registriere ich einen Kundenvertrag im Kundenbereich?
چطور از صفحه مشتری Order ثبت کنم؟
چطور قرارداد مشتری را از تب Orders ثبت کنم؟
```

The assistant must explain the customer-scoped flow accurately and must not only recommend the old Operations & Planning / Orders & Planning Records route.

## Important Domain Clarification

In this product, this flow may be described by users as:

```text
contract
customer contract
order
customer order
Auftrag
Kundenauftrag
Vertrag
Plan
Einsatzplanung
```

But the technical implementation is not a single standalone `Contract` module and not a single monolithic `Plan` table.

The customer-scoped flow orchestrates these existing aggregates:

```text
Customer
  → Customer Order
  → Order equipment lines
  → Order requirement lines
  → Order documents/attachments
  → Planning Record
  → Planning Record documents/attachments
  → Shift Plan
  → Shift Series
  → Shift Series Exceptions
  → Generated concrete shifts
  → Staffing Coverage handoff
```

The Assistant must communicate this distinction clearly when needed.
