# AI-45 — German/English RAG Prompt Assembly, Source-Basis Response, and Golden Evaluation

## Purpose

Ensure that OpenAI receives enough SicherPlan-specific context and is instructed to answer as a grounded domain expert, not as a generic model.

This task focuses on the final RAG prompt assembly, source-basis handling, and evaluation with German/English golden questions.

---

## Product Requirement

The final user-facing answer must be:

- generated by OpenAI;
- grounded in retrieved SicherPlan sources;
- in the same language as the user;
- precise about what is verified;
- honest about missing context;
- supported by source basis;
- free of invented UI labels, workflows, routes, or permissions.

This task is limited to **German and English**.

---

## Prompt for Codex

You are working on the SicherPlan repository.

Implement **AI-45 — German/English RAG Prompt Assembly, Source-Basis Response, and Golden Evaluation**.

### Step 1 — Inspect Prompt Builder

Inspect:

```text
backend/app/modules/assistant/prompt_builder.py
backend/app/modules/assistant/openai_client.py
backend/app/modules/assistant/schemas.py
backend/app/modules/assistant/service.py
backend/app/modules/assistant/grounding.py
web/apps/web-antd/src/components/sicherplan/assistant/*
```

Report:

2. Whether grounding sources are included.
3. Whether source IDs are included.
4. Whether the model is required to use retrieved context.
5. Whether structured response includes source basis.
6. Whether the frontend renders source basis or evidence.
7. Whether there are tests proving prompt contains retrieved content.

### Step 2 — Update Prompt Requirements

The system/developer prompt must include:

```text
You are the SicherPlan AI Assistant.
You answer only about SicherPlan.
For in-scope questions, answer using the provided RAG grounding context.
Do not invent SicherPlan page names, buttons, fields, workflows, permissions, or data.
If a precise claim is not supported by grounding context, say that it is not verified.
Use exact UI labels only when a verified UI/page-help source provides them.
If the user asks in German, answer in German.
If the user asks in English, answer in English.
Keep technical platform terms when needed.
When context is ambiguous, ask a targeted clarifying question.
Use source_basis to identify which retrieved facts support the answer.
```

### Step 3 — Structured Output Schema

Extend or confirm structured response includes:

```json
{
  "answer": "string",
  "confidence": "low|medium|high",
  "out_of_scope": false,
  "diagnosis": [],
  "links": [],
  "missing_permissions": [],
  "next_steps": [],
  "source_basis": [
    {
      "source_type": "knowledge_chunk|page_help_manifest|workflow_help|api_export|role_page_coverage|diagnostic",
      "source_name": "string",
      "page_id": "string|null",
      "module_key": "string|null",
      "title": "string|null",
      "evidence": "short safe evidence"
    }
  ],
  "rag_trace_id": "uuid|null"
}
```

If `source_basis` already exists, validate and improve it. If not, add it.

### Step 4 — Backend Source-Basis Validation

OpenAI may propose source basis, but backend must validate it.

Rules:
- source basis entries must correspond to retrieved source IDs/types;
- invented sources are removed;
- if model makes precise UI claims, there must be a matching UI/page-help source;
- if no content-bearing source exists, confidence must not be high;
- unauthorized links are removed.

### Step 5 — Frontend Rendering

Update assistant UI to show source basis in a clean way, for example:

```text
Sources used
- PS-01 — Platform Services Workspace
- C-01 — Customers Workspace
- P-02 — Orders & Planning Records
```

Keep it user-friendly. Do not expose debug internals to normal users.

### Step 6 — Golden Question Evaluation

Create a golden evaluation test suite for German and English.

Add tests:

```text
backend/tests/modules/assistant/test_rag_golden_questions_en_de.py
backend/tests/modules/assistant/test_rag_source_basis_validation.py
backend/tests/modules/assistant/test_rag_prompt_quality.py
frontend tests if source basis is rendered
```

Golden questions:

```text
Wie registriere ich einen neuen Vertrag?
How do I register a new contract?
How do I create a new customer?
Wie erstelle ich einen neuen Kunden?
How do I create a new order for a customer?
Wie erstelle ich einen neuen Auftrag für einen Kunden?
How do I assign an employee to a shift?
Wie weise ich einen Mitarbeiter einer Schicht zu?
How do I release a shift to the employee app?
Wie gebe ich eine Schicht für die Mitarbeiter-App frei?
```

Each test must assert:

- provider is called;
- grounding context is attached;
- content-bearing sources are present;
- answer language matches question language;
- source_basis is present;
- no unsupported high-confidence claims;
- no generic answer like "look around these workspaces";
- no standalone Contract page is invented.

### Step 7 — Evaluation Report

Add a developer command or test output that prints a compact evaluation table:

```text

```

This is for development/QA only.

### Acceptance Criteria

This task is complete only when:

- OpenAI prompt includes real grounding content.
- German and English answers use source-grounded facts.
- Source basis is returned and validated.
- Frontend renders source basis.
- Golden questions pass.
- The assistant no longer gives generic unsourced answers for common workflow questions.
- No canned final answers are introduced.
