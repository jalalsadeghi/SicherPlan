# AI-26 — Frontend Tests

## Purpose

Add frontend tests for the SicherPlan AI Assistant web UI, API client, store, route persistence, structured response rendering, navigation links, feedback controls, disabled state, and error handling.

This task validates the frontend behavior delivered by tasks `AI-22`, `AI-23`, and `AI-24`.

The tests must confirm that the assistant widget behaves as a global, persistent, permission-aware UI component and does not break the existing Vben/Vue application layout or routing.

## Implementation Rules

1. Do not call the real backend in frontend unit tests.
2. Do not call OpenAI from frontend tests or production frontend code.
3. Do not add unrelated UI changes.
4. Do not change existing route definitions unless tests reveal a real assistant integration issue.
5. Do not hardcode role/permission assumptions in UI that should come from backend capabilities.
6. Do not render raw JSON to users.
7. Do not use `v-html` for assistant content unless sanitized and already approved by project policy.
8. Do not expose internal IDs unless they are part of the structured response and intended for navigation/debug display.
9. Keep tests deterministic.
10. If a test framework is missing, add the minimal standard setup and document the new command.
