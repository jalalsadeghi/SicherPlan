# AI-62 — Golden QA and Assistant Answer Quality for Customer Orders / Contracts

## Purpose

Validate that the Assistant now answers customer-scoped order/contract questions correctly.

This task adds golden QA tests and answer-quality criteria so future changes do not regress the new Customers → Orders → Order Workspace knowledge.

---

## Non-Negotiable Rule

Do not hardcode final answers. Golden QA criteria must evaluate retrieval, grounding, source basis, and answer concepts, not enforce a single fixed text.

### Step 3 — Expected Answer Criteria

Every answer for customer-scoped order creation must explain:

1. Start in **Customers**.
2. Select/open the customer.
3. Open the **Orders** tab.
4. Use **New order** for a new flow, or **Edit** for an existing order.
5. The hidden/canonical customer-scoped route is **Order workspace**.
6. The wizard steps:
   - Order details;
   - Order scope & documents;
   - Planning record;
   - Planning documents;
   - Shift plan;
   - Series & exceptions.
7. Final action **Generate Series & Continue** creates concrete shifts and leads to **Staffing Coverage**.
8. Contract documents are linked/uploaded through Order documents or Planning documents, not through a fake Contract page.
9. If no customer is selected, the answer must tell the user to select a customer first.
