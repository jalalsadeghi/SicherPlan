# AI-61 — Retrieval, Intent Routing, and Safe Links for Customer-Scoped Order Questions

## Purpose

After AI-60 adds the knowledge, update retrieval and navigation so the Assistant chooses the customer-scoped order flow when the user explicitly asks about creating/registering an order or contract from the customer page.

Current problem:
- The assistant only suggests the older/canonical Operations & Planning route.
- It does not know that the user can start from Customers → selected customer → Orders → New order.
- It does not create safe links to the selected customer’s Orders tab or the Order Workspace.

---

## Non-Negotiable Rule

Do **not** add canned final answers. This task improves routing, retrieval, and safe navigation context only.

### Step 3 — Query Classification Rules

If a question mentions order/contract/plan and customer-page context, prefer the customer-scoped flow.

Signals:

English:
```text
from customer page
from the customer
inside customer
customer tab
customer orders
customer contract
create order for this customer
```

German:
```text
beim Kunden
im Kundenbereich
Kunden-Workspace
Kundenseite
Orders-Tab
Kundenauftrag
Vertrag beim Kunden
Auftrag direkt beim Kunden
```

Persian:
```text
از صفحه مشتری
در بخش مشتری
تب Orders
قرارداد مشتری
Order برای مشتری
ثبت قرارداد از صفحه مشتری
```

Do not replace the canonical planning route for general questions such as:

```text
How do I manage all orders?
Where are Orders & Planning Records?
Wie verwalte ich alle Aufträge?
```

For those, the canonical Operations & Planning / Orders & Planning Records route can still be relevant.

### Step 4 — Route Context Awareness

When the user is currently on:

```text

/admin/customers?customer_id=<id>
/admin/customers/order-workspace?customer_id=<id>
```

the Assistant should treat the selected customer as current page context.

Do not expose raw tenant/user/customer IDs in the answer, but use them internally to build safe links.

### Step 5 — Safe Link Generation

Add safe navigation link generation for:

#### Customers Orders tab

```text
label_en: "Customer Orders tab"
label_de: "Orders-Tab des Kunden"
label_fa: "تب Orders مشتری"
path: /admin/customers?customer_id=<customer_id>&tab=orders
route_name: SicherPlanCustomers
```

Only include `customer_id` when:
- present in current route context;
- allowed by current user scope;
- safe to use.

Fallback without selected customer:

```text
/admin/customers
```

#### Order Workspace

```text
label_en: "Order workspace"
label_de: "Order Workspace"
label_fa: "Order workspace"
path: /admin/customers/order-workspace?customer_id=<customer_id>
route_name: SicherPlanCustomerOrderWorkspace
```

For editing an existing order:

```text
/admin/customers/order-workspace?customer_id=<customer_id>&order_id=<order_id>
```

Only include `order_id` if it came from safe route/tool context.

#### Staffing Coverage handoff

```text
label_en: "Staffing Coverage"
label_de: "Staffing Coverage"
label_fa: "Staffing Coverage"
path: /admin/planning-staffing
route_name: SicherPlanPlanningStaffing
```

Add planning filters only if they are safe/verified from current wizard context.
