Use `docs/sprint/AI-Assistant.md` as the authoritative sprint document for this task.

## 4. Required TypeScript Types

Create or align types for the assistant contract. The exact file location must follow repository conventions, but suggested file:

```text
web/apps/web-antd/src/api/sicherplan/assistant.ts
```

Required frontend types:

```ts
export type AssistantLanguageCode = string;

export type AssistantConfidence = 'low' | 'medium' | 'high';

export type AssistantFindingSeverity = 'info' | 'warning' | 'blocking';

export interface AssistantCapabilities {
  enabled: boolean;
  provider_mode?: 'openai' | 'mock' | string;
  can_chat: boolean;
  can_run_diagnostics?: boolean;
  can_reindex_knowledge?: boolean;
  supported_features?: string[];
}

export interface AssistantRouteContext {
  path: string;
  route_name?: string | null;
  page_id?: string | null;
  query?: Record<string, unknown>;
  ui_locale?: string | null;
  timezone?: string | null;
  visible_page_title?: string | null;
}

export interface AssistantClientContext {
  timezone?: string | null;
  ui_locale?: string | null;
  visible_page_title?: string | null;
}

export interface AssistantDiagnosisItem {
  finding: string;
  severity: AssistantFindingSeverity;
  evidence?: string | null;
}

export interface AssistantLink {
  label: string;
  path: string;
  route_name?: string | null;
  page_id?: string | null;
  reason?: string | null;

export interface AssistantMissingPermission {
  permission: string;
  reason?: string | null;
}

export interface AssistantStructuredResponse {
  conversation_id: string;
  message_id: string;
  detected_language?: AssistantLanguageCode | null;
  response_language?: AssistantLanguageCode | null;
  answer: string;
  scope?: string | null;
  confidence?: AssistantConfidence;
  out_of_scope?: boolean;
  diagnosis?: AssistantDiagnosisItem[];
  links?: AssistantLink[];
  missing_permissions?: AssistantMissingPermission[];
  next_steps?: string[];
  tool_trace_id?: string | null;
}

export interface AssistantMessage {
  id: string;
  role: 'user' | 'assistant' | 'tool' | 'system_summary';
```

Adapt naming to backend JSON response conventions if the implemented API uses camelCase or snake_case. Do not silently mix styles. Use one explicit mapping strategy if needed.

## 7. Persistence Requirements

Persist the following safely:

- `activeConversationId`
- `isOpen`
- `draftInput`

Use localStorage or sessionStorage according to existing project conventions. If no convention exists:

- use localStorage for `activeConversationId` and `isOpen`;
- use sessionStorage for `draftInput`.

Do not persist sensitive tool outputs separately from the backend conversation. Do not persist API keys, auth tokens, or raw backend errors.

Use a namespaced key, for example:

```text
sicherplan.assistant.activeConversationId
sicherplan.assistant.isOpen
sicherplan.assistant.draftInput
```

## 8. Route Context Builder

Implement a helper that collects the current route context without treating it as authoritative.

Required fields where available:

```ts
{
  path,
  route_name,
  page_id,
  query,
  ui_locale,
  timezone,
  visible_page_title
}
```

Rules:

- `path` should come from Vue Router.
- `route_name` should come from route name if available.
- `page_id` should come from route meta if available; otherwise null.
- `query` should be shallow and serializable.
- `ui_locale` should come from the current frontend i18n state if easily available.
- `timezone` may use `Intl.DateTimeFormat().resolvedOptions().timeZone`.
- `visible_page_title` may be route meta title if available.

The backend must still validate all access independently.

## 9. Error Handling

Normalize API errors into user-friendly messages for the UI. Do not expose stack traces, raw provider errors, API keys, JWTs, or internal exception objects.

Suggested structure:

```ts
interface AssistantErrorState {
  message: string;
  code?: string;
  status?: number;
}
```

At store level, keep `error` as a safe string or normalized object based on project conventions.

## 10. Security Constraints

- Do not call OpenAI from the browser.
- Do not add frontend-only permission logic as a substitute for backend authorization.
- Do not expose hidden fields from structured responses.
- Do not store raw sensitive tool payloads in localStorage.
- Do not create write-action API calls.
- Do not add any assistant action that creates, updates, releases, approves, deletes, archives, or dispatches business records.
