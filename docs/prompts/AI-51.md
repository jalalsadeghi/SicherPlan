# AI-51 — Collapsible “Sources used” Section in Assistant Messages

## Purpose

The assistant currently renders `Sources used` directly under every answer. This is useful for QA and trust, but it makes the chat panel crowded for normal users.

We want the source section to remain available but hidden by default. The user should be able to toggle it open/closed by clicking the `Sources used` heading.

---

## Current Repository Context to Verify

Codex must inspect current files before implementation:

```text
web/apps/web-antd/src/components/sicherplan/assistant/AssistantMessageList.vue
web/apps/web-antd/src/components/sicherplan/assistant/AssistantEvidenceList.vue
web/apps/web-antd/src/api/sicherplan/assistant.ts
web/apps/web-antd/src/locales/*
```

Known current behavior:
- `AssistantMessageList.vue` renders `source_basis` whenever it exists.
- The section is always visible.
- The title is passed as `sourcesTitle`.
- The current layout makes the panel crowded.

---

## Requirements

1. `Sources used` must be hidden by default for every assistant message.
2. The heading/button `Sources used` must always be visible when `source_basis` exists.
3. Clicking `Sources used` expands the section.
4. Clicking it again collapses the section.
5. Expansion/collapse must happen below the answer; the main answer text must not be rewritten or rerendered.
6. The message should remain readable in the small fixed assistant panel.
7. The toggle must be keyboard accessible.
8. The toggle must expose `aria-expanded`.
9. The section must preserve existing source data.
10. Source basis should still be available for tests and QA.
11. Default collapsed state should apply when loading past conversations.
