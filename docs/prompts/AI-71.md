# AI-71 — Deterministic Field/Lookup Corpus Export and Stable Artifact Freshness Check

## Purpose

The GitHub Actions job no longer fails because of missing dependencies. The export command now runs and prints:

```text
field_count=219
lookup_count=10
field_counts_by_module={'planning': 38, 'customers': 106, 'platform_services': 6, 'employees': 69}
lookup_counts_by_module={'customers': 6, 'planning': 4}
warnings=['fa_locale_missing']
```

The current failure is caused by the artifact freshness check:

```text
diff -u app/modules/assistant/generated/field_lookup_corpus.json /tmp/tmp....
Error: Process completed with exit code 1.
```

The diff shows two important problems:

1. The committed artifact is stale because source hashes changed:
   - `backend_modules`
   - `frontend_root`

2. The export output is likely not fully deterministic because `source_basis` entries appear in a different order between the committed artifact and regenerated output.

This task must make the field/lookup corpus export deterministic and then refresh the committed artifact.

---

## Current Evidence

The failed CI diff includes examples like:

```diff
- "CustomerNewPlanWizardState includes field planning_mode_code."
+ "CustomerDashboardPlanItemRead includes field planning_mode_code."
```

and source-basis entries moving position, for example:

```diff
- "CustomerDashboardCalendarItemRead includes field starts_at."
...
+ "CustomerDashboardCalendarItemRead includes field starts_at."
```

The diff also shows source hashes changed:

```diff
- "backend_modules": "371ff..."
+ "backend_modules": "6d2b..."
- "frontend_root": "8ec5..."
+ "frontend_root": "a568..."
```

This means the generated artifact does not match the committed artifact.

---

## Non-Negotiable Requirements

1. Do not remove the artifact freshness check.
2. Do not make CI pass by ignoring the diff.
3. Do not hardcode only the current changed lines.
4. Make the export deterministic.
5. Refresh and commit the generated artifact after deterministic export is fixed.
6. Ensure repeated generation produces byte-identical JSON.
7. Keep `Vertragsreferenz`, `Rechtlicher Name`, and other known field-label questions working.
8. Keep unrelated terms such as `Apfelkuchen` out-of-scope.

### Step 5 — Make Field and Source-Basis Ordering Deterministic

Before serialization, normalize and sort every list that can affect JSON output.

Sort:

```text
field_definitions by field_key
lookup_definitions by lookup_key
source_basis by (source_type, source_name, page_id, module_key, evidence)
aliases
label_keys
binding_paths
schema_fields
route_names
form_contexts
related_fields
labels per language
lookup values
matched values if serialized
```

Also remove duplicates deterministically.

Suggested helper:

```python
def _sort_source_basis(items: list[FieldSourceBasis]) -> list[FieldSourceBasis]:
    unique = {
        (
            item.source_type,
            item.source_name,
            item.page_id or "",
            item.module_key or "",
            item.evidence,
        ): item
        for item in items
    }
    return [
        unique[key]
        for key in sorted(unique)
    ]
```
