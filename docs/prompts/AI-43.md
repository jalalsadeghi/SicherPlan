# AI-43 — German/English Workflow Knowledge and Source-Basis Model

## Purpose

Create structured, retrievable workflow knowledge for SicherPlan so OpenAI can answer workflow questions like a domain expert without relying on canned responses.

This task focuses on **workflow facts and source-basis metadata** for German and English questions. It must not generate final answers itself.

---

## Product Requirement

When the user asks a workflow question, the assistant must retrieve a precise workflow context.

Examples:

```text
How do I create a customer?
Wie erstelle ich einen neuen Auftrag für einen Kunden?
How do I assign an employee to a shift?
Wie gebe ich eine Schicht für die Mitarbeiter-App frei?
Wie registriere ich einen neuen Vertrag?
```

The assistant must know which workflow is involved, which pages are relevant, what preconditions exist, what permissions may be needed, and which source facts support the answer.

### Step 4 — Source Basis

Every workflow step must include source basis metadata.

Source basis must identify:

```text
source_type
source_name
page_id
module_key
evidence
```

If evidence is inferred from existing source docs, label it as such. Do not fabricate exact UI labels.

### Step 5 — Workflow Retrieval Tool

Update or add a tool:

```text
assistant.search_workflow_help
```

It must return workflow facts, not final answers.

Inputs:

