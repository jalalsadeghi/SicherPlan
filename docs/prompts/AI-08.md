## Goal

Implement the language detection and response-language policy required by the SicherPlan AI Assistant.

The assistant must respond in the same language used by the user. This applies to normal answers, placeholder answers, diagnostic summaries, permission messages, out-of-scope refusals, link explanations, and future provider/tool-based responses.

This task must establish the backend language policy and integrate it into the structured response contract from AI-07. It must not implement OpenAI, knowledge retrieval, diagnostic tools, or frontend widget behavior.

## Mandatory Language Rule

The assistant must answer in the same language used by the user.

Examples:

- German question -> German answer.
- Persian question -> Persian answer.
- English question -> English answer.
- Another detectable language -> answer in that language when reasonably detectable.
- Mixed-language question -> answer in the dominant language, while preserving technical product terms.
- Technical terms may remain unchanged when translation would reduce clarity.

Technical terms that may remain unchanged:

```text
Tenant

Actual Record

```

### 2. Detection Strategy

Use a lightweight deterministic detection strategy suitable for tests and early implementation.

Minimum required behavior:

#### Persian / Farsi

Detect as `fa` when the text contains a meaningful amount of Persian/Arabic script characters, especially Persian-specific characters:

```text
ا ب پ ت ث ج چ ح خ د ذ ر ز ژ س ش ص ض ط ظ ع غ ف ق ک گ ل م ن و ه ی
```

Also handle common Persian words:

```text
من
چطور
چگونه
مشکل
کارمند
شیفت
نمایش
پلتفرم
```

#### German

Detect as `de` when the text contains German-specific signals, for example:

```text
ä ö ü ß
ich
wie
warum
nicht
Mitarbeiter
Schicht
angezeigt
freigeben
```

#### English

Detect as `en` for primarily English text or as fallback when the text is Latin-script and does not strongly match German or another supported language.

#### Ambiguous / Empty

- Empty text should be rejected by existing validation before language detection if possible.
- If detection is ambiguous, prefer the latest user message language if available.
- If still ambiguous, use `ui_locale` when present.
- If no signal exists, use `unknown` or project-default language, but the structured response must still include language fields.

### 3. Dominant Language for Mixed Text

Implement a simple dominance policy:

- If Persian script is present in significant amount, prefer `fa`.
- If German signals are stronger than English signals, prefer `de`.
- Otherwise prefer `en` for Latin-script text.
- Preserve technical terms unchanged.

Example:

```text
"من برای Markus یک Shift assigned کردم ولی در App نمایش داده نمی‌شود"
```

Expected language:

```text
fa
```
